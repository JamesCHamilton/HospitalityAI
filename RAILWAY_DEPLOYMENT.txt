HOSPITALITY AI - RAILWAY DEPLOYMENT GUIDE

To deploy your two Python services (Agent and ETL) on Railway, follow these steps.

---

### 1. Repository Preparation
Railway needs to find your dependencies. Ensure you have a 'requirements.txt' file in your 'backend/' folder or at the root of the repository.

Content of requirements.txt:
fastapi
uvicorn
httpx
psycopg2-binary
sqlalchemy
python-dotenv
openai
python-multipart
unsiloed-sdk

---

### 2. Deployment Strategy (Two Services)
In Railway, you will create TWO separate services from the same GitHub repository.

#### SERVICE A: AI Reasoning Engine (Agent)
- Root Directory: backend/agent
- Start Command: uvicorn main:app --host 0.0.0.0 --port $PORT
- Public Domain: Generated by Railway (e.g., agent-production.up.railway.app)

#### SERVICE B: Ingestion Gateway (ETL)
- Root Directory: backend/etl
- Start Command: uvicorn main:app --host 0.0.0.0 --port $PORT
- Public Domain: Generated by Railway (e.g., etl-production.up.railway.app)

---

### 3. Required Environment Variables
You must add these variables in the Railway dashboard under the "Variables" tab for EACH service:

- user: (Your Supabase user)
- password: (Your Supabase password)
- host: (Your Supabase host)
- port: 5432
- dbname: postgres
- OPENAI_API_KEY: (Your OpenAI Key)
- UNSILOED_API_KEY: (Your Unsiloed Key)
- CRUSTDATA_API_KEY: (Your Crustdata Key)

---

### 4. Step-by-Step Dashboard Setup
1. Go to Railway.app and click "New Project".
2. Select "Deploy from GitHub repo" and choose your HospitalityAI repo.
3. Once the first service is created, go to **Settings > General > Root Directory** and set it to `backend/agent`.
4. Add a second service by clicking "New" in your project view, choose the same repo, and set its Root Directory to `backend/etl`.
5. For the **ETL Service**, go to **Settings > Build & Deploy** and ensure the Start Command is: `uvicorn main:app --host 0.0.0.0 --port $PORT`.
6. For the **Agent Service**, ensure the Start Command is: `uvicorn main:app --host 0.0.0.0 --port $PORT`.
7. Generate a Public Domain for both services in the **Settings** tab so your frontend can reach them.

---

### 5. Frontend Update
Once deployed, remember to update the URLs in your Next.js frontend ('frontend/app/patient/page.tsx' and 'frontend/app/insurance/page.tsx'):

Change:
'http://localhost:8080' -> 'https://your-agent-service.up.railway.app'
'http://localhost:8081' -> 'https://your-etl-service.up.railway.app'

---

### 💡 Pro Tip:
Railway automatically injects the 'PORT' variable. By using '--port $PORT' in your start command, your FastAPI app will correctly listen to the external world.
